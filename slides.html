Robotic Operating System

ROS Introduction

Robotic middleware

A Robot is a combination of different compnents :

  • Hardware components :
    • Computer
    • Motor controllers
    • Sensors
  • Software components :
    • OS
    • Hardware drivers
    • Robot servoing
    • Perception
  • Problem :
    • Task to solve
    • Can be distributed (multi-robots, multi-PC)
  • Integration between differents layers : middleware
What is ROS

ROS stands for Robot Operating System

  • Open source robotics middleware
  • Robotics framework organized around a common communication library
  • Packaging system with dependence & compilation management
  • Large community
  • Thousand of existing modules
What ROS is not
  • Not an operating system : use Ubuntu (official) or other experimentals platforms (MS Windows, OS X, Fedora, Android, etc.)
  • Not a programming language :
    • Official : C++, Python
    • Experimental : Java, Lisp, Octave, etc.
  • Not a real time environment
  • Not designed for embedded systems (need to use rosserial bridge for this purpose)

ROS Concepts

Structure

Structure
  • Ros Application = set of communicating processes
  • Node = a process that used the ROS framework
  • Different Nodes may reside in the same or different machines transparently
  • Nodes get to know one another via rosmaster

Communication mechanism

Communication mechanism

Two main communication protocols :

  • Messages
    • Data flow, message passing : asynchronous
    • Grouped in topics
  • Services
    • Remote procedure call, synchronous
    • Request / Answer messages
    •

    Both are staticaly typed
