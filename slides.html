Introduction to ROS

Motivation for Robotic Middleware

A Robot is a composition of different components

  • Hardware components
    • Computational Units (FPGA, PC, Microcontroller, …)
    • Actuators (Motor, …)
    • Sensors (Camera, IR, Telemeter, …)
  • Software components
    • Hardware drivers
    • Robot servoing
    • Perception

Motivation for Robotic Middleware

Motivation for Robotic Middleware

A Robotic application is an interconnection of components

  • Heterogenous components
    • Cross-language
    • Cross-platform
  • Distributed components
    • Multi-robots
    • Multi-PC
    • Multi-network
  • Dynamic components
    • Appear, Disappear (mobility)
    • Crash, Evolve

Both Robots and a Robotic Applications are assemblage of many other people's components

Need for a Robotic Operating System

What is (not) ROS

What is ROS

ROS is a 'Meta' Operating System for Robotics Applications

  • An open source robotics framework
    • Maintained by Willow Garage (PR2, TutrleBot, ROS, OpenCV, PCL, …)
    • Cross-platform (Ubuntu, OSx, Windows, Android, …)
    • Cross-language (C/C++, Python, Java, …)
  • An architecture for distributed inter-process/inter-machine communication and configuration
    • Object like message wrapping/unwrapping
    • Message passing (Publisher/Subscriber, RPC)
    • Graph node communication with master node
    • Centralized configuration
  • A collection of tools
    • packaging, dependency/build management, shell introspection, …
    • runtime tools, data analysis, graphic user interfaces, …
  • A Large community with thousand (and counting) of existing modules
What ROS is not
  • Not an operating system : use Ubuntu (official) or other experimentals platforms (MS Windows, OS X, Fedora, Android, etc.)
  • Not a programming language :
    • Official : C++, Python
    • Experimental : Java, Lisp, Octave, etc.
  • Not a real time environment
  • Not designed for embedded systems (need to use rosserial bridge for this purpose)

ROS Concepts

Structure

Structure
  • Ros Application = set of communicating processes
  • Node = a process that used the ROS framework
  • Different Nodes may reside in the same or different machines transparently
  • Nodes get to know one another via rosmaster

Communication mechanism

Communication mechanism

Two main communication protocols :

  • Topics
    • Data flow, message passing : asynchronous
  • Services
    • Remote procedure call : synchronous
    • Request / Answer messages

Both are staticaly typed : Messages & Services definition

Topics
  • Directory used for topics discovery : ROS master;
  • Multiple concurrent publishers / subscribers for a single topic;
  • Publish / subscribe order irrelevant.
Services
  • Directory used for service discovery : ROS master;
  • Multiple simultaneous clients
ROS master
  • Directory for publisher / subscribers/ services;
  • XMLRPC API;
  • Not a central communication node
  • part of roscore
Roscore
  • roscore :
    • rosmaster,
    • parameter server,
    • log aggregator.
  • Parameter server :
    • centralized parameter repository : provides parameter access to all nodes,
    • XMLRPC data type.
Roslaunch
  • Automate startup of complex ROS applications ;
  • Launch files specify :
    • list of nodes,
    • with topics / service remapping,
    • parameter values,
    • XML syntax.
Packages and Stacks
  • Packages :
    • atomic unit of building,
    • can contain anything : nodes, messages, tools, launch files, etc.
    • in their most basic form :
      • package_name
      • package_name/Makefile
      • package_name/CMakeLists.txt
      • package_name/manifest.xml
  • Stacks :
    • atomic unit of "releasing",
    • collection of packages grouped together.
    • in their most basic form :
      • stack_name
      • stack_name/package_name_1
      • stack_name/package_name_n
      • stack_name/stack.xml
Transformation frames

Robot:

  • set of rigit bodies linked together;
  • each sensor obtain measurement in its local reference.

In ROS

  • set of transformation frames,
  • linked by transformation,
  • organized in a directed tree,
  • published on a single topic /tf,
  • each sensor data is associated to a frame.

ROS tools

Runtime inspection

Node inspection :

  • rosnode : list nodes and interconnections,
  • rxgraph : connection graph;

Message inspection :

  • rostopic : list topics, print sent messages,
  • rosservice : list services.
Logging

Logging API :

  • different verbosity levels (info, debug, error, etc.),
  • published on an unique topic /rosout,
  • rxconsole for online inspection,
  • automatic dumping to file system for offline analysis.
Data visualization

rviz

  • full 3D visualization,
  • display almost all standard sensor messages and frames,
  • plugin system to handle build-in messages.
Data recording

Recording messages :

  • rosbag record : generic subscriber,
  • container : bagfile,

Replaying messages :

  • rosbag play : generic publisher,
  • rxbag graphic interface.
Thid party modules

Hardware drivers :

  • sensors : almost all common used robotics sensors,
  • robots : depends on constructors.

Software stacks :

  • SLAM (localization and mapping),
  • navigation,
  • 3D perception,
  • simulation;

Growing community.

Getting Started with ROS

Create new package
  1. Create an empty package :
    • $ roscreate-pkg pkg_name
  2. Edit pkg_name/manifest.xml : 
    <package>
      <description brief="pkg_name">
         My package description
      </description>
      <author>Mr Bob</author>
      <license>BSD</license>
      <review status="unreviewed" notes=""/>
      <url>http://ros.org/wiki/pkg_name</url>
    </package>
  3. Edit pkg_name/CMakeLists.txt : 
    #uncomment if you have defined messages
    #rosbuild_genmsg()
    #uncomment if you have defined services
    #rosbuild_gensrv()

    #common commands for building c++ executables and libraries
    #rosbuild_add_library(${PROJECT_NAME} src/example.cpp)
    #target_link_libraries(${PROJECT_NAME} another_library)
    #rosbuild_add_boost_directories()
    #rosbuild_link_boost(${PROJECT_NAME} thread)
    #rosbuild_add_executable(example examples/example.cpp)
    #target_link_libraries(example ${PROJECT_NAME})
